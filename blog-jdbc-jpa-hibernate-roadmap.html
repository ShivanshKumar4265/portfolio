The JDBC → JPA → Hibernate Roadmap

A Chronological, Connected Learning Path for Java Database Mastery.

Back to Portfolio
How to read this map: Every layer builds on the previous one. JDBC is the raw foundation. JPA is the abstraction standard built on top of JDBC's concepts. Hibernate is the most powerful implementation of JPA — and understanding JDBC first makes Hibernate feel logical, not magical.
Phase 1 JDBC (Java Database Connectivity)
Talk to the database the raw, manual way
▼

JDBC is Java's lowest-level API for database communication. You write SQL yourself, manage connections yourself, and map results yourself. It's verbose — but understanding it makes everything above it click.

1.1 Foundations & Setup
Core Concepts
  • What is JDBC? — The Java standard API for relational database access (java.sql)
  • JDBC Architecture — DriverManager → Connection → Statement → ResultSet
  • JDBC Drivers — Type 1, 2, 3, 4 drivers; why Type 4 (thin driver) is used today
  • Adding a JDBC Driver — Maven dependency (e.g., mysql-connector-java)
  • DriverManager vs DataSource — Differences, when to use which
1.2 Establishing a Connection
Connections
  • DriverManager.getConnection() — The basic connection call
  • JDBC URL format — jdbc:mysql://host:port/dbname?params
  • Connection properties — Timeouts, SSL, charset
  • Connection lifecycle — Open, use, close
  • Why connections are expensive — TCP handshake, auth, state
1.3 Executing SQL & 1.4 Reading Results
Statements
  • Statement — Simple SQL execution
  • PreparedStatement — Parameterized queries; prevents SQL injection
  • CallableStatement — Calling stored procedures
  • executeQuery() vs executeUpdate() vs execute()
  • Batch execution — addBatch() / executeBatch()
ResultSets
  • ResultSet — Cursor-based iteration over rows
  • ResultSet.next() — Row navigation
  • Column access — getString(), getInt(), etc.
  • ResultSetMetaData — Inspect column names, types dynamically
  • Scrollable & Updatable — TYPE_SCROLL_INSENSITIVE
1.5 Transactions & 1.6 Pooling
Transactions
  • auto-commit mode — Default ON
  • setAutoCommit(false) — Start a manual transaction
  • commit() / rollback()
  • Savepoints — Partial rollbacks
  • Isolation levels — Read Uncommitted to Serializable
  • Dirty read, Non-repeatable read, Phantom read
Connection Pooling
  • The problem — Creating connection per request is slow
  • What a pool does — Reuses pre-created connections
  • HikariCP — The fastest modern JDBC pool
  • Pool configuration — maximumPoolSize, idleTimeout
  • DataSource interface — Standard way to get pooled connections
JDBC Pain Points (Why We Need More): Manual SQL, manual mapping (row by row), manual try-catch-finally, no object relationships, no caching, no lazy loading. These pain points are exactly what JPA and Hibernate solve.
Phase 2 JPA (Jakarta Persistence API)
A standard contract for Object-Relational Mapping
▼

JPA is not a library — it's a specification. It defines how Java objects should be mapped to database tables, how queries should be written, and how transactions should be managed — without dictating the implementation.

2.1 ORM Concept & 2.2 Entity Mapping
The ORM Concept
  • What is ORM? — Object-Relational Mapping
  • The impedance mismatch — Objects vs Tables
  • JPA vs JDBC — JPA sits above JDBC
  • JPA Providers — Hibernate, EclipseLink
Core Annotations
  • @Entity, @Table
  • @Id, @GeneratedValue (AUTO, IDENTITY, SEQUENCE)
  • @Column, @Transient
  • @Enumerated, @Temporal, @Lob
2.3 Relationships & 2.4 Fetch Types
Relationships
  • @OneToOne, @OneToMany
  • @ManyToOne, @ManyToMany
  • @JoinColumn, @JoinTable
  • mappedBy — The non-owning side
  • Owning side vs inverse side concept
Fetching & Cascades
  • FetchType.EAGER vs LAZY
  • The N+1 problem
  • CascadeType (PERSIST, MERGE, REMOVE, ALL)
  • orphanRemoval = true
2.6 Entity Lifecycle & 2.7 EntityManager
Entity States
  • Transient — New object
  • Managed (Persistent) — Tracked by EntityManager
  • Detached — Was managed, now un-tracked
  • Removed — Marked for deletion
EntityManager API
  • persist(), find()
  • merge(), remove()
  • flush(), refresh()
  • detach(), clear()
2.9 JPQL & 2.10 Criteria API
JPQL
  • Operates on entities, not tables
  • SELECT e FROM Employee e
  • JOIN FETCH (solves N+1)
  • Named parameters :paramName
  • @NamedQuery
Criteria API & Tx
  • Type-safe, programmatic queries
  • CriteriaBuilder, CriteriaQuery
  • @Transactional (Spring)
  • Propagation types & Rollback rules
  • Inheritance Strategies (SINGLE_TABLE, JOINED)
Phase 3 Hibernate & Spring Data
The most powerful JPA implementation and modern tools
▼

Hibernate implements JPA, but also has its own powerful features that go beyond the spec. This is where performance, caching, and advanced mapping become your tools.

3.1 Architecture & 3.3 Persistence Context
Core Hibernate
  • SessionFactory (= EntityManagerFactory)
  • Session (= EntityManager)
  • hbm2ddl.auto (update, create-drop)
  • Dialects (MySQLDialect)
First-Level Cache
  • A per-Session identity map
  • Identity guarantee
  • Dirty checking — Auto-detects changes
  • Flush → SQL generation timing
3.4 HQL & 3.6 Caching
Advanced Queries
  • HQL vs JPQL — HQL is a superset
  • createNativeQuery(sql)
  • Solving N+1: @BatchSize, EntityGraph
  • @FetchProfile
Caching Strategy
  • First-Level (L1) — Always on
  • Second-Level (L2) — Shared across Sessions
  • L2 Providers (Ehcache, Redis)
  • Query Cache — Caches IDs, not entities
3.8 Advanced Mapping & 3.11 Locking
Advanced Mapping
  • @Formula — Computed columns
  • @Where — Soft delete filtering
  • @Filter, @Immutable
  • Collections: @ElementCollection
Locking & Batching
  • Optimistic locking — @Version
  • Pessimistic locking — FOR UPDATE
  • Batch Processing — hibernate.jdbc.batch_size
  • StatelessSession for pure ETL
3.16 Spring Data JPA
The Practical Layer
  • JpaRepository<T, ID> — CRUD + paging
  • Derived query methods — findByEmailAndAge()
  • @Query for custom JPQL
  • Pageable / Page<T> / Slice<T>
  • Specification<T> — Composable predicates
  • @Modifying + @Transactional for bulk updates
  • Auditing (@CreatedDate, @LastModifiedDate)

The Full Connection Map

  RAW JDBC
  ─────────────────────────────────────────────────────
  DriverManager → Connection → Statement → ResultSet
  Manual SQL | Manual Mapping | Manual Transactions
       │
       │  Pain: verbose, error-prone, no object awareness
       ▼
  JPA SPECIFICATION (Standard)
  ─────────────────────────────────────────────────────
  @Entity/@Table → EntityManager → JPQL → @Transactional
  Standard contract any provider must fulfill
  Still generates SQL and uses JDBC underneath
       │
       │  JPA is the "what". Hibernate is the "how".
       ▼
  HIBERNATE (JPA Provider + Extensions)
  ─────────────────────────────────────────────────────
  SessionFactory/Session → L1/L2 Cache → HQL/Criteria
  Batch processing | Envers | Filters | Custom types
  Generates optimized SQL → sends via JDBC → DB
       │
       │  Spring Data JPA wraps Hibernate for productivity
       ▼
  SPRING DATA JPA
  ─────────────────────────────────────────────────────
  JpaRepository → Derived queries → Specifications
  @Transactional | @EnableJpaAuditing | Projections
