DDL — Data Definition Language

DDL commands define and modify the structure of database objects (databases, tables, views, indexes, triggers, routines). In MySQL, DDL commands are auto-committed immediately — no ROLLBACK is possible after execution.

CREATE DATABASE

Creates a new database schema. You can optionally specify character sets and collations to handle Unicode data correctly.

MySQL 
CREATE DATABASE IF NOT EXISTS ecommerce
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

Under the Hood: Character Sets

utf8 vs utf8mb4: In MySQL, the legacy utf8 character set represents utf8mb3 (maximum of 3 bytes per character). It cannot store 4-byte characters such as emojis or specific rare Chinese scripts. Always prefer utf8mb4 in production schemas.

CREATE TABLE & Constraints

Creates a physical table layout. Defines columns, data types, and structural integrity constraints.

MySQL 
CREATE TABLE IF NOT EXISTS orders (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  total DECIMAL(10, 2) DEFAULT 0.00,
  status ENUM('pending', 'completed', 'refunded') DEFAULT 'pending',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id) 
    ON DELETE CASCADE 
    ON UPDATE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
Foreign Key Action Effect on Child when Parent row changes
ON DELETE CASCADE Automatically deletes all matching child rows in the child table.
ON DELETE SET NULL Sets the foreign key column in all child rows to NULL.
ON DELETE RESTRICT / NO ACTION Prevents deletion of parent row if child rows refer to it. (Throws SQL error).

Interview Trap: CHECK & ENUM Constraints

  • CHECK: Before MySQL 8.0.16, CHECK constraints were silently parsed but ignored entirely by the engine. Always specify MySQL versions when discussing CHECK constraint security in interviews!
  • ENUM: Appending a new value in the middle of an ENUM column list triggers a full table copy/rebuild, blocking concurrent reads. Adding it at the end of the ENUM is online and extremely fast.

Generated Columns (MySQL 5.7.6+)

Generated columns are computed expressions built directly into the table structure.

MySQL 
CREATE TABLE rectangles (
  width DOUBLE,
  height DOUBLE,
  area DOUBLE AS (width * height) VIRTUAL,      -- Computed on-the-fly when read
  perimeter DOUBLE AS (2 * (width + height)) STORED -- Saved on disk, computed on write
);

ALTER TABLE & Schema Changes

Modifies column layouts, properties, or table names. Can target columns or indices.

MySQL 
-- Add index in-place with zero read/write block (Online DDL)
ALTER TABLE users ADD INDEX idx_status(status), ALGORITHM=INPLACE, LOCK=NONE;

-- Modify column type (rebuilds table)
ALTER TABLE users MODIFY COLUMN name VARCHAR(255) NOT NULL;

Production Schema Upgrades at Scale

Running native ALTER TABLE on tables with 100M+ rows block writes. In enterprise high-concurrency systems, always use external online migration utilities such as pt-online-schema-change (Percona) or gh-ost (GitHub) to prevent lockups.

2

DML — Data Manipulation Language

DML commands manipulate row values inside the schema layout. These statements participate in transactions, meaning their mutations are transactional and can be ROLLED BACK.

INSERT & Variants

Inserts new records into tables. Batch insertions are highly optimized.

MySQL 
-- 1. Optimized Multi-row Batch Insert
INSERT INTO users (name, email) VALUES
  ('Alice', 'alice@gmail.com'),
  ('Bob', 'bob@gmail.com'),
  ('Charlie', 'charlie@gmail.com');

-- 2. INSERT ... ON DUPLICATE KEY UPDATE (Upsert)
INSERT INTO page_views (page_id, views) VALUES (101, 1)
ON DUPLICATE KEY UPDATE views = views + 1;

-- 3. REPLACE INTO (DELETE if exists + INSERT new row)
REPLACE INTO users (id, name, email) VALUES (1, 'Alice Updated', 'new@gmail.com');

REPLACE INTO vs Upsert (ON DUPLICATE KEY UPDATE)

REPLACE INTO: If a primary key exists, it triggers a physical DELETE followed by a new INSERT. This increments the table auto-increment ID, drops all foreign-key dependencies under CASCADE, and fires both AFTER DELETE and AFTER INSERT triggers! Always use ON DUPLICATE KEY UPDATE to update fields in-place safely.

UPDATE with JOINs

Mutates existing records based on complex relationships.

MySQL 
UPDATE orders o
JOIN users u ON o.user_id = u.id
SET o.discount = 0.10
WHERE u.tier = 'gold' AND o.status = 'pending';

DELETE with LIMIT Batching

Deletes records matching search criteria.

MySQL 
-- Highly optimized batch deleting to prevent long locking
DELETE FROM logs 
WHERE created_at < DATE_SUB(NOW(), INTERVAL 30 DAY) 
ORDER BY created_at ASC 
LIMIT 5000;

Under the Hood: DELETE Lock Overhead

Running a massive DELETE FROM logs WHERE age > 365 acquires row locks on millions of records, bloats the InnoDB Undo Log, and blocks critical application threads. Instead, run deletes in small batches using LIMIT or partition the tables horizontally so you can run an instant, zero-cost DROP PARTITION.

3

DQL — Data Query Language

DQL (consisting of the SELECT command) is the engine for data retrieval. It uses logical processing phases and cost-based optimizations to return records.

Logical Query Execution Order

In SQL, the order in which a query is parsed and executed is completely different from how it is written:

Step Phase Logical Action Description
1 FROM / JOIN Retrieves rows from referenced physical or logical tables. Sets up Cartesian layouts.
2 WHERE Filters rows before grouping based on literal condition checks.
3 GROUP BY Combines filtered row results into distinct bucket sets.
4 HAVING Filters aggregate buckets (calculated values) after grouping is complete.
5 SELECT / DISTINCT Calculates projections, executes CASE conditions, maps aliases, and drops duplicates.
6 ORDER BY Sorts the final projected tuples based on ASC/DESC settings.
7 LIMIT / OFFSET Discards rows up to OFFSET and returns a maximum count of LIMIT rows.

Interview Trap: Where Aliases Fail

Because the WHERE clause is processed at Step 2, before the SELECT clause (Step 5), you cannot refer to projected field aliases inside your filter checks!
-- ERROR: SELECT salary * 12 AS annual_sal FROM emp WHERE annual_sal > 60000;

Common Table Expressions & Recursive Queries

CTEs provide a highly readable, modular way to write complex subqueries.

MySQL 
-- Recursive CTE to traverse an organizational reporting structure
WITH RECURSIVE org_tree AS (
  -- Anchor Member: Root Manager
  SELECT id, name, manager_id, 1 AS level
  FROM employees 
  WHERE manager_id IS NULL
  
  UNION ALL
  
  -- Recursive Member: Join children to parents
  SELECT e.id, e.name, e.manager_id, ot.level + 1
  FROM employees e
  JOIN org_tree ot ON e.manager_id = ot.id
)
SELECT * FROM org_tree ORDER BY level;

Window Functions

Window functions perform analytics across sets of rows linked to the current record without combining them.

MySQL 
-- Running totals and relative change analysis
SELECT 
  date, 
  revenue,
  -- 1. Running Total
  SUM(revenue) OVER (ORDER BY date ROWS UNBOUNDED PRECEDING) AS running_total,
  -- 2. Previous Month Value
  LAG(revenue, 1, 0) OVER (ORDER BY date) AS prev_month_revenue,
  -- 3. Dense Rank
  DENSE_RANK() OVER (ORDER BY revenue DESC) AS revenue_rank
FROM monthly_sales;

Performance Tip: Deep Pagination Scan Trap

Running queries with large offsets (e.g. LIMIT 1000000, 10) forces MySQL to scan and discard 1,000,000 index rows, causing extreme performance degradation. Always use Keyset/Cursor-based Pagination:
SELECT * FROM sales WHERE id > last_seen_id ORDER BY id LIMIT 10;

DCL — Data Control Language

DCL handles user authentication permissions, role authorizations, and schema access control.

MySQL 
-- Create User with secure authentication
CREATE USER 'readonly_app'@'%' IDENTIFIED WITH caching_sha2_password BY 'StrongP@ss123';

-- Grant specific permissions on a database
GRANT SELECT, INSERT ON ecommerce.* TO 'readonly_app'@'%';

-- Column-level grant security controls
GRANT SELECT (id, name, email) ON ecommerce.users TO 'support_agent'@'localhost';

-- Apply changes immediately
FLUSH PRIVILEGES;

TCL — Transaction Control Language & ACID

TCL commands manage database transactions to ensure data consistency across multiple statements.

MySQL 
START TRANSACTION;
  UPDATE accounts SET balance = balance - 500.00 WHERE id = 1;
  SAVEPOINT step_one;
  
  UPDATE accounts SET balance = balance + 500.00 WHERE id = 2;
  -- Oops, invalid account number. Roll back to savepoint:
  ROLLBACK TO SAVEPOINT step_one;
  
COMMIT;

ACID Isolation Levels vs Anomaly Matrices

Database engines use isolation levels to manage concurrent access anomalies.

Isolation Level Dirty Reads Non-Repeatable Reads Phantom Reads
READ UNCOMMITTED Possible Possible Possible
READ COMMITTED Prevented Possible Possible
REPEATABLE READ (Default) Prevented Prevented Prevented (via MVCC)*
SERIALIZABLE Prevented Prevented Prevented

Under the Hood: MVCC & Phantom Reads

Under the default REPEATABLE READ isolation level, InnoDB uses MVCC (Multi-Version Concurrency Control) to prevent phantom reads on snapshot reads (standard SELECT statements) by serving historical row states. However, for current reads (e.g. SELECT ... FOR UPDATE), InnoDB prevents phantom reads by acquiring next-key locks (combining row-level locks with gap locks in indexes).

JOIN Masterclass

Joins combine columns from multiple tables using shared primary and foreign key values.

MySQL 
-- 1. INNER JOIN (Only matching rows)
SELECT u.name, o.total FROM users u JOIN orders o ON u.id = o.user_id;

-- 2. LEFT JOIN (All left, matching right or NULL)
SELECT u.name, o.total FROM users u LEFT JOIN orders o ON u.id = o.user_id;

-- 3. FULL OUTER JOIN Simulation (MySQL workaround)
SELECT u.name, o.total FROM users u LEFT JOIN orders o ON u.id = o.user_id
UNION
SELECT u.name, o.total FROM users u RIGHT JOIN orders o ON u.id = o.user_id;

Under the Hood: Join Algorithms

  • Nested Loop Join: Outer loops drive inner database index checks. Highly efficient for small tables or when keys are indexed.
  • Hash Join (MySQL 8.0.18+): Builds an in-memory hash map of the smaller table, then scans the larger table to find matches. Extremely fast for large, unindexed datasets.

Index Commands & B+ Tree

Indexes speed up query execution by providing fast data lookup paths.

MySQL 
-- Create Composite Covered Index
CREATE INDEX idx_user_status_total ON orders (user_id, status, total);

-- Create Prefix Index (For long text columns)
CREATE INDEX idx_email_prefix ON users (email(20));

Leftmost Prefix Rule Traps

For a composite index on (a, b, c):
- Filters on a or (a, b) or (a, b, c) can use the index.
- Filters on b or c or (b, c) cannot use the index! MySQL must skip the index and perform a full table scan.

Stored Procedures

Stored procedures bundle business logic directly into the database engine, reducing network overhead.

MySQL 
DELIMITER $$
CREATE PROCEDURE transfer_funds(
  IN from_account INT,
  IN to_account INT,
  IN amount DECIMAL(10,2)
)
BEGIN
  DECLARE EXIT HANDLER FOR SQLEXCEPTION
  BEGIN
    ROLLBACK;
    SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = 'Transfer failed due to system error';
  END;

  START TRANSACTION;
    UPDATE accounts SET balance = balance - amount WHERE id = from_account;
    UPDATE accounts SET balance = balance + amount WHERE id = to_account;
  COMMIT;
END $$
DELIMITER ;

Built-in Functions

Built-in functions perform standard transformations on strings, dates, numbers, and JSON types.

MySQL 
-- 1. Date Formats & Math
SELECT DATE_FORMAT(NOW(), '%Y-%m-%d %H:%i:%s') AS formatted_now;

-- 2. JSON Functions (MySQL 8)

Triggers

Triggers execute automatically when DML modifications occur on a target table.

MySQL 
CREATE TRIGGER after_salary_update
AFTER UPDATE ON employees
FOR EACH ROW
BEGIN
  IF OLD.salary != NEW.salary THEN
    INSERT INTO salary_audit(emp_id, old_salary, new_salary, changed_at)
    VALUES (OLD.id, OLD.salary, NEW.salary, NOW());
  END IF;
END;

Views & WITH CHECK OPTION

Views are saved query templates that look like virtual tables. Using the WITH CHECK OPTION prevents users from inserting rows that would not match the view's query filters.

MySQL 
CREATE OR REPLACE VIEW active_products AS
  SELECT id, name, stock FROM products 
  WHERE stock > 0
  WITH CHECK OPTION;

-- Throws an error because stock=0 violates the stock > 0 WHERE clause check!
INSERT INTO active_products(id, name, stock) VALUES (99, 'Product X', 0);

Event Scheduler

Allows you to schedule tasks on a recurring or one-off basis, similar to cron jobs.

MySQL 
-- Enable the global background process thread
SET GLOBAL event_scheduler = ON;

-- Schedule a recurring daily cleanup event
CREATE EVENT daily_session_cleanup
ON SCHEDULE EVERY 1 DAY STARTS '2026-05-24 02:00:00'
ON COMPLETION PRESERVE
DO 
  DELETE FROM temp_sessions WHERE expired_at < NOW();

Metadata Commands

Retrieves info about database schemas, connections, and engine runtime metrics.

MySQL 
SHOW FULL PROCESSLIST; -- View active query threads and states
SHOW ENGINE INNODB STATUS; -- Heavy performance metrics & locks
SHOW CREATE TABLE users; -- View physical DDL details

Locking Commands

Controls locks explicitly inside transactions to prevent race conditions during updates.

MySQL 
START TRANSACTION;
  -- Exclusive Lock (Prevents any read/write from other transactions)
  SELECT * FROM accounts WHERE id = 1 FOR UPDATE;
  
  -- Shared Lock (Allows other reads but blocks modifications)
  SELECT * FROM rates WHERE id = 12 FOR SHARE;
COMMIT;

Import / Export

Provides methods for bulk data migration, loading external files, or dumping schemas.

Bash / SQL 
# CLI Export: Dump entire database
mysqldump -u root -p ecommerce > backup.sql

# SQL Import: Bulk import CSV at high speed (Bypasses parsing)
LOAD DATA INFILE '/var/lib/mysql-files/users.csv'
INTO TABLE users
FIELDS TERMINATED BY ',' OPTIONALLY ENCLOSED BY '"'
LINES TERMINATED BY '\n'
IGNORE 1 ROWS;
